これまでの対話をまとめ、開発の指針となる情報を整理・要約しました。

1. サービスコンセプト

「曖昧なプロンプトを、AIとの対話を通じて最強のプロンプトへ昇華させる」
ユーザーの入力に対してAIが不足情報を診断し、追加質問への回答を経て、構造化された最適なプロンプトを自動生成する自分専用ツールです。

2. ユーザーフロー（6ステップ）

1.  入力: ユーザーが「ブログを書いて」などの曖昧な依頼を入力。
2.  分析: AIが「役割・対象・制約・形式・トーン」の観点から不足項目を特定。
3.  質問: 特定された不足項目に基づき、AIがユーザーへ2〜3問の質問を提示。
4.  回答: ユーザーがフォーム形式で補足情報を入力。
5.  生成: 「元の依頼＋回答」を統合し、構造化された改善プロンプトを出力。
6.  完了: 改善理由の確認と、クリップボードへのコピー。

3. 技術スタック・設計のポイント

  - フロントエンド: Next.js (App Router) + Tailwind CSS + Lucide-React。
  - バックエンド: Next.js API Routes をプロキシとして使用し、CORS回避とAPI統合を管理。
  - マルチモデル対応:
      - Gemini API: クラウド経由での高速・高精度な処理。
      - LM Studio: OpenAI互換APIを利用したローカルLLMでの実行。
  - 汎用設計: AIモデルごとの差異を Provider レイヤーで吸収し、将来的なモデル追加（Claude, Ollama等）も容易な構造。
  - JSON出力の安定化: AIにJSON形式で回答させることで、動的な質問フォーム生成を実現。

4. 推奨ファイル構造

src/
├── app/
│   ├── api/chat/route.ts  # API中継・モデル切り替えロジック
│   └── page.tsx           # メインUI（状態遷移とフォーム管理）
├── constants/
│   └── prompts.ts         # 分析・改善用のシステムプロンプト定義
├── types/
│   └── ai.ts              # ProviderやAPIレスポンスの型定義
└── lib/                   # 共通処理ロジック

5. 差別化とメリット

  - 「対話型」の自動化: 単なる変換ではなく、不足分を「聞く」ことで、誰でもプロンプトエンジニア級の出力が得られる。
  - プライベート利用に最適化: APIキーの管理やローカルLLMの活用により、コストを抑えつつ自由にカスタマイズ可能。
  - プロンプトスキルの向上: 「改善のポイント」を表示することで、自分自身がどのような情報を書くべきかの学習にもなる。

結論：
このプロジェクトは、**「プロンプトエンジニアリングという技術的な壁を、AIとの対話というUIで解消する」**非常に実用的で拡張性の高いツールです。まずは提供したプロトタイプコードをベースに、自分の好みのプロンプトテンプレート（prompts.ts）を育てることから始めるのが最短ルートです。
